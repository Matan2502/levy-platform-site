3 מה קרה כאן? (5 המושגים)

  • Endpoint (כתובת)הכתובת ב-URL שאליה פנינו — api.frankfurter.dev/v1/latest
  • Method (פעולה)השתמשנו ב-GET = "תביא לי מידע" (זו ברירת המחדל של fetch)
  • Query parametersמה שמופיע אחרי ה-? — from=USD&to=ILS — אלה הפרמטרים שבחרת
  • JSON responseהשרת החזיר טקסט בפורמט JSON, והקוד הפך אותו לאובייקט עם response.json()
  • Status code200 = הצלחה. קודים אחרים (401, 404, 500) מספרים שמשהו השתבש

ועכשיו — חשבוניות לרשות המיסים

ה-API של רשות המיסים עובד בדיוק ככה: אותו fetch, אותו JSON שחוזר. ההבדלים היחידים — הבקשה תהיה POST (שולחים פרטי חשבונית במקום רק לקרוא מידע), יתווסף Authorization עם token (כי זה מידע רגיש), ובמקום rates.ILS תקרא את השדה allocationNumber (מספר ההקצאה). אותו עיקרון — שכבת אבטחה אחת מעל.
